Coherence-Loss Regularizer + Latency-of-Resolution probes

Companion empirical probes. CLR penalizes models for inconsistent coherence states across paraphrased prompts; LOR measures how long it takes a model to fully resolve a multi-part question. Together they target the open question of whether DCS scores correlate with broader alignment behavior.

Code

The full v8.7 Go implementation lives in work/dcs-gate/. The entry point is main.go (routing) and the most interesting files are judge.go (judge prompting + parsing), analyzer.go (deterministic stage), baseline.go (corpus + similarity), stream_demo.go (streaming endpoint embedded HTML), and filter.go (security-redaction filters applied to streamed content). ~3,000 lines of Go, 51 tests passing.

Reproducibility

The 20 / 30 / 72 smoke tests were run on a Google Colab T4 (free tier). The notebook does the full setup — install Ollama, pull mxbai-embed-large and qwen3:14b, download the v8.7 binary, run health check, run the three contrastive tests, capture all outputs — in one pass. All output cells are preserved in the committed version so you can audit the runs even before reproducing.
